# Hellovia Translate — Flutter (`phone/`)

A **Flutter** rewrite of the Expo/React-Native app in `../mobile`. Same screens,
same server, same Socket.IO contract — just Dart instead of React Native. It talks
to the same backend at `https://translate.hellovia.app` (or your local dev server).

> This folder contains only the Dart source (`lib/`), assets, and `pubspec.yaml`.
> The native `android/` and `ios/` projects are generated by `flutter create`
> (step 2). Flutter is **not** yet installed on this machine — start at step 1.

## 2. Generate the native projects

This project ships only `lib/`, `assets/`, and `pubspec.yaml`. Generate the
`android/` and `ios/` folders without touching them.

**Safe method (recommended)** — scaffold in a throwaway dir and move the native
folders in, so nothing here is overwritten:

```bash
cd /Users/dave/projects/live-translate
flutter create --org app.hellovia --project-name phone _scaffold
mv _scaffold/android _scaffold/ios phone/
rm -rf _scaffold
cd phone
flutter pub get
```

**Quick method** — `flutter create .` adds platform folders to an existing
project. It generally leaves `lib/` and `pubspec.yaml` alone, but may regenerate
template files like `.gitignore` / `analysis_options.yaml`:

```bash
cd /Users/dave/projects/live-translate/phone
flutter create . --org app.hellovia --project-name phone --platforms=android,ios
flutter pub get
```

## 3. Native configuration

### Android — `android/app/src/main/AndroidManifest.xml`

Add permissions (inside `<manifest>`, above `<application>`):

```xml
<uses-permission android:name="android.permission.INTERNET"/>
<uses-permission android:name="android.permission.RECORD_AUDIO"/>
<uses-permission android:name="android.permission.READ_MEDIA_IMAGES"/>
```

For a **local http dev server** (e.g. `http://10.0.2.2:4000`), also allow
cleartext on the `<application>` tag (remove for production / https):

```xml
<application
    android:usesCleartextTraffic="true"
    ... >
```

### Android — OAuth callback scheme (`android/app/build.gradle`)

`flutter_web_auth_2` needs the redirect scheme registered. Inside
`android { defaultConfig { ... } }`:

```gradle
manifestPlaceholders += [ 'appAuthRedirectScheme': 'hellovia-translate' ]
```

### Android — applicationId

To match the existing Play Store listing, set in `android/app/build.gradle`:

```gradle
defaultConfig {
    applicationId "app.hellovia.translate"
    ...
}
```

### iOS — `ios/Runner/Info.plist`

```xml
<key>NSMicrophoneUsageDescription</key>
<string>Hellovia Translate uses your microphone to record voice messages for live translation.</string>
<key>NSPhotoLibraryUsageDescription</key>
<string>Hellovia Translate uses your photo library to set your profile picture.</string>
```

## 4. Point the app at your server

The server URL is a compile-time constant (`lib/config.dart`). It **defaults to
production** (`https://translate.hellovia.app`), overridable via `--dart-define`
for local development:

```bash
# production (default — no flag needed)
flutter run

# local dev: Android emulator → server on your Mac's localhost:4000
flutter run --dart-define=SERVER_URL=http://10.0.2.2:4000

# local dev: physical device → your Mac's LAN IP
flutter run --dart-define=SERVER_URL=http://192.168.x.x:4000
```

`10.0.2.2` is the Android emulator's alias for the host machine. Cleartext
(`http://`) is only needed for local dev — see the `usesCleartextTraffic` note in
step 3; production over `https://` doesn't need it.

### Using a `.env` file (recommended for dev)

Instead of passing flags every time, put your overrides in `.env` (already
provided; gitignored — `.env.example` is the committed template) and load it with
Flutter's built-in `--dart-define-from-file` (no package needed):

```env
# phone/.env
SERVER_URL=https://translate.hellovia.app
REQUIRE_AUTH=false      # skip the Google sign-in gate during dev
```

```bash
flutter run --dart-define-from-file=.env
```

The keys map to `fromEnvironment()` constants in `lib/config.dart`
(`SERVER_URL` → `kServerUrl`, `REQUIRE_AUTH` → `kRequireAuth`). To require login
again, set `REQUIRE_AUTH=true` (or just run without the flag — the default).

## 6. Build for the Play Store

```bash
flutter build appbundle --dart-define=SERVER_URL=https://translate.hellovia.app
# output: build/app/outputs/bundle/release/app-release.aab
```

Signing, versioning and the Play Console upload steps are the same as documented
in `../mobile/README.md` (App Bundle → Internal testing → Production). Set
`version: 1.0.0+1` in `pubspec.yaml` (the `+1` is the Android `versionCode`).

## Architecture notes

- **State**: a single `AppState` (`ChangeNotifier`) holds the UI language, the
  loaded translation bundles, and the persisted `UserPrefs`. It's exposed through
  a small `InheritedNotifier` so any widget can call `context.appState.t('key')`
  and rebuild when the language changes. This replaces react-i18next + the
  `useUserPrefs` hook.
- **Socket**: `SocketService` is a lazily-created singleton mirroring
  `socket.ts` (manual connect, polling→websocket, 5 reconnect attempts). Room
  create/join/peek use `emitWithAck`; messaging uses fire-and-forget `emit`.
- **Voice**: hold the mic to record AAC/m4a via `record`, base64-encode, and
  emit `message:audio` — identical payload shape to the RN app
  (`{ audioBase64, mimeType: 'audio/m4a', durationMs }`). Clips under 1s are
  rejected client-side.
- **Translations**: keys are looked up by dot-path against the JSON bundles in
  `assets/locales/`, with `{{param}}` interpolation and an English fallback.
